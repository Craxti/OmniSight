# Сопоставление ТЗ конкурса с реализацией OmniSight РСМ

Источник требований: [`Требования_к_РСМ_конкурс.docx`](./Требования_к_РСМ_конкурс.docx) · текстовая копия: [`_requirements_extract.txt`](./_requirements_extract.txt).

**Версия API:** каноническая реализация — **`/api/v1`**, как в §8 ТЗ. UI, автоприёмка и OpenAPI используют v1.

**Автоприёмка:** `cd apps/api && python scripts/seed_demo.py && python scripts/verify_demo_acceptance.py` (20 проверок, FR 1–54 и §8).

## §9 — нефункциональные требования

| Категория | Статус | Реализация |
|-----------|--------|------------|
| Производительность resolve/search | ✅ | NFR &lt;200 мс, `tests/test_nfr_scale.py` (опц. 50k CI) |
| Масштабируемость | ✅ | индексы, пагинация, кэш PostgreSQL, rate limit |
| Надёжность / целостность | ✅ | валидация связей, soft-delete (`is_deleted`) |
| Логирование | ✅ | `RequestLoggingMiddleware`, JSON-логи в production |
| OpenAPI | ✅ | FR 43 |
| Асинхронный API (read/write pools) | ✅ | 100% async HTTP: native ORM + `session.run_sync` bridge, без thread-pool DB |
