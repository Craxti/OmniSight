# Сценарий демо для жюри

**Русская версия** · [English](DEMO_GUIDE.md)

Пошаговый сценарий демонстрации OmniSight РСМ на демо-данных PAY.  
Шпаргалка с формулировками для выступления: [`ПРЕЗЕНТАЦИЯ.md`](./ПРЕЗЕНТАЦИЯ.md)

## Подготовка

```powershell
# Из корня репозитория — UI + API + Demo CE
.\scripts\start-all.ps1
```

| Сервис | URL | Учётные данные |
|--------|-----|----------------|
| Web UI | http://localhost:5173 | `admin@omnisight.local` / `admin123` |
| API / Swagger | http://localhost:8000/docs | X-API-Key: `omnisight-api-key-dev` |
| Demo CE | http://localhost:8090 | — |

Демо-модель: **11 CI**, **10 связей** (платёжный контур PAY).

---

## Часть 1 — Обзор платформы (3 мин)

1. Вход → **Обзор** (`/`) — KPI: 11 CI, 10 связей, зелёный Model health.
2. **Элементы** (`/inventory`) — реестр CI с внешними ID (hostname, IP, serviceCode).
3. **Зависимости** (`/relations`) — все **7 типов** связей ТЗ; кнопка **Проверить модель**.
4. **Карта** (`/graph`) — выбрать `demo-biz` или `demo-db`; показать business path и панель impact.

**Ключевой тезис:** OmniSight — не мониторинг и не correlation engine, а **источник правды** о зависимостях ИТ.

---

## Часть 2 — Корреляция A: объединить алерты (5 мин)

**Вопрос:** *Четыре алерта за минуту — один инцидент или четыре?*

1. Открыть **Корреляция** (`/correlation`).
2. Ввести 4 строки (в каждой одно поле):

| Строка | Поле | Значение | Попадает в |
|--------|------|----------|------------|
| 1 | hostname | `app-01` | demo-app |
| 2 | ip | `10.0.0.5` | demo-db |
| 3 | externalId | `ext-db-1` | demo-db |
| 4 | serviceCode + applicationCode | `PAY` + `PAY-APP` | demo-biz |

3. Нажать **Обработать** (Ingest).

**Ожидаемый результат:**
- **Сопоставлено = 4**
- **Связанная цепочка = true** (зелёным)
- **Зона первопричины** — `demo-db`
- Граф с подсветкой цепочки

**Для CE:** `chain_related: true` → один INC.

---

## Часть 3 — Корреляция B: не объединять (3 мин)

**Вопрос:** *Два алерта в одном контуре — всегда один инцидент?*

1. Очистить ввод, ввести 2 строки (только hostname):

| Строка | hostname | CI | Слой |
|--------|----------|-----|------|
| 1 | `pay-srv-01` | demo-pay-srv | инфраструктура (`hosted_on`) |
| 2 | `app-01` | demo-app | приложение (`depends_on` → БД) |

2. Нажать **Обработать**.

**Ожидаемый результат:**
- **Сопоставлено = 2**
- **Связанная цепочка = false** (жёлтым)
- Блока зоны первопричины нет

**Для CE:** `chain_related: false` → два отдельных INC.

---

## Часть 4 — Autodiscover (3 мин)

**Проблема:** у `demo-app` есть hostname, но **нет IP** — алерт только по IP не сопоставится.

1. **Элементы** → открыть `demo-app` — убедиться, что IP нет.
2. **Autodiscover** → галочка **`demo-pay-srv`** → **Ручной режим** → **Сканировать**.
3. Вкладка **Поля:** `demo-app` / `ip` / **`10.0.0.10`**
4. Вкладка **Связи:** `depends_on` → `demo-db` (если предложено).
5. **Применить выбранное** → IP появился у `demo-app`.
6. **Аудит** (`/audit`) — запись с diff.

**Ключевой тезис:** модель актуальна без ручного CMDB — человек подтверждает данные с сервера.

---

## Часть 5 — Интеграция Demo CE (опционально, 2 мин)

1. Открыть http://localhost:8090
2. Отправить 4 тестовых алерта → показать результат ingest от РСМ.
3. В UI: **Настройки → API** — `X-API-Key` и ссылка на Swagger.
